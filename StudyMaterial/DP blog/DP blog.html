DP BLOG :

In this section , we will look at some of the advanced concepts under dynamic programming and would also learn their applications in few problems .

Let us first start what should we observe in a problem in order to find a dp solution to it . In our greedy algorithms blog , we saw that a greedy problem should have optimal substructure and greedy choice property . In some problems , optimal substructure exists but there is no greedy choice property meaning there is no way for us to know what can be best first move without looking at all the inputs , rather what we have are overlapping sub-problems , that is what we can find a solution to a bigger solution using the solution to an already solved smaller sub-problem .

Combining both these ideas , we can generalise and say that if we can somehow convert how problem into a DAG (Directed A-cyclic graph) of sub-problems , we would be able to solve it using dp in polynomial time .

Let us see what it means :

Q) Why should the graph of sub-problems be directed ?

  • We showed earlier there should be optimal substructure meaning an optimal solution can be constructed from the optimal solutions of its sub-problems . That is why graph should have directed edges so that we know how we reached at the current state as this would allow us to calculate the answer for the current state using the solutions for the previously calculated sub-problems .

Q) Why should the graph not contain any cycles ?

  • Consider a directed graph with three vertices and edges like

    1−>2,2−>3,3−>11->2,2->3,3->11−>2,2−>3,3−>1﻿ , so in order to find solution for sub-problem 3 , we should know solution for problem 2 for which in turn we need solution solution for problem 1 . But there exists an edge 3−>13->13−>1﻿ , that is we cannot know solution for problem 1 without knowing solution for problem 3 which is the problem we originally wanted to solve . Hence if the graph of sub-problems contains a cycle we cannot solve it using dp since we would reach a deadlock like here . Hence the graph should be acyclic so that optimal sub-structure is preserved.

Implementation :

int find_lis(vector<int> a) {
	vector<int> dp;
	for (int i : a) {
		int pos = lower_bound(dp.begin(), dp.end(), i) - dp.begin();
		if (pos == dp.size())  // we can have a new, longer increasing subsequence!
			dp.push_back(i);
		else  // oh ok, at least we can make the ending element smaller
			dp[pos] = i;
	}
	return dp.size();
}

DP with Bitmasks :

This concept can be used to solve the problems where we need to iterate over all the subsets of the input .

Let us first understand what is a bitmask ?

A bitmask is a representation of subsets of a given set , so if we want to represent a subset of this set then it can be encoded by a sequence of N bits where the i'th bit is set (1) if A[i]A[i]A[i]﻿ is a part of our set and 0 if it is not . For example , consider a set of numbers A= {1,2,3,4,5} , we can represent any subset of A using a bitmask of length 5 for like 100101001010010﻿ would represent the set {A[1],A[4]A[1],A[4]A[1],A[4]﻿} since those bits are set in our mask .

Now how does dp come into the picture ?

Again using the optimal sub-structure and overlapping sub-problem property , we can assign a value to each mask (that is to each subset) and compute the values for the new masks using already computed values .

Formally what we try to do is we try to find the value for a subset A , we remove an element in every possible way and use values for obtained subsets A'1, A'2, ... , A'k to compute the value for A.

This means that the values for Ai' must have been computed already, so we need to establish an ordering in which masks will be considered. It's easy to see that the natural ordering will do: go over masks in increasing order of corresponding numbers.

This technique can be used to solved various problems . Get ready to apply this technique in the quiz section .
