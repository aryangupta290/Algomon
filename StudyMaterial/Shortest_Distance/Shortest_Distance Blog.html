Bellman Ford :

Bellman Ford algorithm similar to dijkstra also allows us to find the shortest path from a vertex to all other vertices of a weighted graph but it can also work with graphs in which edges can have a negative weights .

If there are negative weights , then they can create a negative weight cycle such that the cycle would reduce the total path distance by coming back to the same point . So algorithms like Dijkstra won't work in these cases which is why we need to study Bellman because while Dijkstra looks only to the immediate neighbours of a vertex, Bellman goes through each edge in every iteration.

The algorithm proceeds as follows :

  • We choose a starting vertex and assign infinity path values to all other vertices .
  • Now we visit each edge and relax the path distances if they are inaccurate .
  • We repeat this V times since in the worst case a vertex's path length might need to be readjusted V times .
  • After all the vertices have their path lengths , we check if a negative cycle is present by going through all edges one more time , and if we get a shorter path , then there is a negative weight cycle .

The time complexity of Bellman ford is O(VE)O(VE)O(VE)﻿ which is more than that of Dijkstra.
