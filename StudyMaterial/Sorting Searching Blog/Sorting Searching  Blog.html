Radix Sort :

This is an advanced sort that allows us to do better than the traditional nlognnlognnlogn﻿ bound (used in counting sort algorithms like merge sort ) .

The idea is we sort the input digit by digit from least significant digit to the most significant digit . So we first group the individual digits of the same place value and then sort the elements according to increasing and decreasing order .

The algorithm proceeds as follows :

  • Find the maximum element in the input array and also the number of digits in the maximum element . Let it be d .
  • Now we go through each significant place one by one and use any stable sorting algorithm like counting sort to sort these digits .
  • Say d was 2 , so we after sorting the digits based on one's place , we will now sort the digits according to ten's place and then finally hundreds place .

For example consider : {121,432,564,23,1,45,788}

The maximum number among then is 788 and the no of digits in it is 3 so d=3 . Using radix sort , if we try to sort these numbers , it would go as follows :

The time complexity to do this would be O(d(n+k))O(d(n+k))O(d(n+k))﻿ where

O(n+k)O(n+k) O(n+k)﻿ is the complexity of counting sort .

The pseudo code for radix sort would look as follows :

radixSort(array)
  d <- maximum number of digits in the largest element
  create d buckets of size 0-9
  for i <- 0 to d
    sort the elements according to ith place digits using countingSort

countingSort(array, d)
  max <- find largest element among dth place elements
  initialize count array with all zeros
  for j <- 0 to size
    find the total count of each unique digit in dth place of elements and
    store the count at jth index in count array
  for i <- 1 to max
    find the cumulative sum and store it in count array itself
  for j <- size down to 1
    restore the elements to array
    decrease count of each element restored by 1

Ternary Search :

We all know about binary search which is used for monotonic functions that is whether the function is either increasing or decreasing .

The idea we use there is we search the sorted array by repeatedly dividing the search interval in half . If the value of the search key is less than the item in the middle of the interval, we narrow the interval to the lower half. Otherwise, narrow it to the upper half. We repeat this procedure until the value is found or the interval is empty .

Ternary search is also quite similar with binary search but instead of monotonic functions , it allows us to search in functions which are uni-modal , that is

  • Either the function strictly increases first, reaches a maximum and then strictly decreases.
  • The function strictly decreases first, reaches a minimum, and then strictly increases.

Let the consider a concave function ( convex function case will also be identical ) .

Our task is to find the maximum of function f(x)f(x)f(x)﻿ on the interval [l,r][l,r][l,r]﻿ .

Consider any 2 points m1 and m2 in this interval such that they follow this inequality : l<m1<m2<rl<m1<m2<rl<m1<m2<r﻿ . Now we evaluate the value of the function at these two intervals . So there can be following two possibilities :

  • f(m1)=f(m2)f(m1)=f(m2)f(m1)=f(m2)﻿ → This means both points belong to area where value of the function is maximised or m1 is in the area of increasing values and m2 is in the area of descending values . Therefore the search space will now be [m1,m2][m1,m2][m1,m2]﻿ .
  • f(m1)>f(m2)f(m1)>f(m2)f(m1)>f(m2)﻿ → Since the maximum cannot be on the right side of m2 , the interval is reduced to [l,m2][l,m2][l,m2]﻿.
  • f(m2)>f(m1)f(m2)>f(m1)f(m2)>f(m1)﻿ → Like before , the maximum cannot be on the left side of m1 , so the interval is reduced to [m1,r][m1,r][m1,r]﻿.

So the algorithm proceeds just like binary search but it allows more flexibility in terms of the functions where we can search for the key . 

int ternarySearch(int array[], int left, int right, int x)
{
   if (right >= left) {
     int intvl = (right - left) / 3;
     int leftmid = left + intvl;
     int rightmid = leftmid + intvl;
     if (array[leftmid] == x)
        return leftmid;
     if (array[rightmid] == x)
        return rightmid;
     if (x < array[leftmid]) {
       return ternarySearch(array, left, leftmid, x);
     }
     else if (x > array[leftmid] && x < array[rightmid]) {
       return ternarySearch(array, leftmid, rightmid, x);
     }
     else {
       return ternarySearch(array, rightmid, right, x);
     }
   }
   return -1;
}

Q 1) We are given the arrival and leaving times of n customers and we need to find the maximum number of customers at any time .

Sample input :

Say n is 3 and the arrival and departure time are as follows :

5 8

2 4

3 9

We can clearly see answer will be 2 for this (1st and 3rd customer )

So to solve this problem , let us first try to differenciate between arrival and departure . We realise that when a person arrives , we will add one to our set and when a person leaves we will subtract one from the set .

Hence it makes sense to mark the arrival times to +1 and departure times to -1 .

Now we can just do pair sorting to find the maximum amount of arrival times of the customers (by maintaining a counter which would add the second value of the pair as we iterate through the sorted array and try to update the max value of cur found so far ) .

cur → denotes the number of people in the mall currently ( that is when we are considering the current person ) 

cin>>n;
vector<int> A;
for(int i=0;i<n;i++)
{
	int x,y;
	cin>>x>>y;
	A.PB({x,1});
	A.PB({y,-1});
}
int ans =0;
int cur =0;
sort(A.begin(),A.end());
for(int i=0;i<2*n;i++)
{
	cur+=A[i].second.
	ans=max(ans,cur);
}
cout<<ans<<endl;

Q 2)Given an array A of n integers , we can perform the following operation on it k times where we take any element of the array and increase it by 1 and our task is to find the maximum possible median .

We can sort the array in ascending order and then binary search on the answer that is the maximum possible median . We realise that the number of operations to raise the median would increase monotonically as x increases so we can use binary search . So for a given median value x , the number of operations required can be found by iterating through the array and hence the time complexity would be O(nlogn).O(nlogn).O(nlogn).﻿

int lastTrue(int lo, int hi, function<bool(int)> test) {
	lo--;
	while (lo < hi) {
		int mid = lo + (hi - lo + 1) / 2;
		if (test(mid)) {
			lo = mid;
		} else {
			hi = mid - 1;
		}
	}
	return lo;
}
