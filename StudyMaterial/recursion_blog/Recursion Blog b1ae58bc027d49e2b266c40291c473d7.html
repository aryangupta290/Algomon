What is recursion?

Well, google says that the meaning of the word "recursion" is "the repeated application of a recursive procedure or definition". This is possibly the simplest way to define a recursive algorithm as well. A recursive algorithm is one which calls itself. For example:

def recursiveFunction(int n):
	if  n==1:
		print 1
		return
	else:
		print n
		n = n-1
		recursiveFunction(n)

The above function is a recursive function that calls itself. It is a function which can be called from some other function in order to print all the natural numbers from n to 1 (in descending order). The above is a basic example of a recursive function.

Recursion to obtain all subsets of a set A Recursive algorithm is very useful to generate all possible subsets of a given set. This is because we know that for each element in a set of n elements, we have 2 choices, either to consider it or not consider it. This is exactly the essence of a recursive algorithm. At each step, either we decide to select something/perform a task or choose to not do it. Thus we have 2 choices at each step and in this manner we can form all the subsets of the given set.

Let us write a basic algorithm to print all sub-strings of a given string str.

void powerSet(string str, int index = -1, string curr = "")
{
    int n = str.length();
    if (index == n)         // this is the base case when we are done with all cases
        return;
    // print current subset
    cout << curr << "\n";

    // now for each character in the string, we have 2 choices. Either to include
    // or not include. The include case id handled when we append that character
    // and pass the string to next recursive call. otherwise not.
    for (int i = index + 1; i < n; i++) {
 
        curr += str[i];
        powerSet(str, i, curr);
        curr.erase(curr.size() - 1);
    }
    return;
}

In this way we can generate all subsets or sub-strings. This is a popular method to solve problems which involve finding an answer from all subsets of a given set. This is often optimized by applying dynamic programming to avoid re-calculation of overlapping sub-problems about which we shall discuss later.

When do we use recursive algorithms?

Recursive algorithms are useful is a variety of cases. Let us consider the problems involving hierarchies or networks or graph. In the algorithmic world, when we talk about graphs, we actually mean a network. A network which involves certain nodes which are connected by edges.

Let us consider a problem to find the number of employees working under the given employee in a company's hierarchy.

In the hierarchy, if we were to find the number of employees working under a given employee, we find the number of employees under each employee working immediately under the employee under consideration and then find its sum to obtain the total number of employees working under a particular person.

You can find the above problem here:

https://cses.fi/problemset/task/1674

We are given the parent of each node of a rooted tree, and we want to compute the sub-tree size for each node. A sub-tree is composed of a root node and the sub-trees of the root's children. Thus, the size of a sub-tree is one plus the size of the root's children' sub-trees. Thus we can formulate a recursive algorithm which finds the solution to all child nodes for a given node by calling itself for the child nodes and then aggregating the results to get the answer for each parent node.
