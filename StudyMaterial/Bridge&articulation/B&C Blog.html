Bridges and Articulation points :

Let us understand the concept of Bridges and Articulation points .

Let us first understand what a DFS tree is since it is going to help us in understanding how to find bridges and Articulation point .

We can form the DFS tree by just doing a simple DFS and maintaining a boolean visited array .

The reason why it is so useful is that the back edges of the graph all connect a vertex with its descendant in the spanning tree .

Suppose that there is an edge u→v, and without loss of generality the depth-first traversal reaches u while v is still unexplored. Then either of the following two possibilities would be true :

  • If the depth-first traversal goes to v from u using u→v, then u→v is a span-edge.
  • If the depth-first traversal doesn't go to v from u using u→v, then v was already visited when the traversal looked at it . Thus it was explored while exploring one of the other neighbours of u, which means that v is a descendant of u in the DFS tree.

Bridges :

An edge in an un-directed connected graph is a bridge if removing it disconnects the graph.A bridge is also known as cut edge . These represent vulnerabilities in the graph which can easily be exploited hence it is important to study them .

For example , in the above graph , the edge from 1→9 is a bridge since removing it increases the number of connected components .

First of all , a bridge would only be an edge which is a part of the DFS tree since if we select some other edge(u→v) , we can always travel through the edges of the DFS tree from u to v and hence such an edge can't be a bridge .

Now for the span edges , we can say that → 2. A span-edge u→v is a bridge if and only if there exists no back-edge that connects a descendant of u→v with an ancestor of u→v. In other words, a span-edge u→v is a bridge if and only if there is no back-edge that "passes over" u→v.

This is because :

Removing the edge u→v splits the spanning tree to two disconnected parts: the subtree of u→v and the rest of the spanning tree. If there is a back-edge between these two components, then the graph is still connected, otherwise u→v is a bridge. The only way a back-edge can connect these components is if it connects a descendant of u→v with an ancestor of u→v.

Now the challenge is how do we check this :

We can define a dp[]dp[] dp[]﻿ array where dp[u]dp[u]dp[u]﻿ denotes the number of back-edges passing over the edge between u and its parent .

dp[u]dp[u]dp[u]﻿=(# of back-edges going up from u)−(# of back-edges going down from u)+∑ dp[v]dp[v]dp[v]﻿ where v is a child of u.

To understand this:

Let u be a vertex and p be its parent. We want dp[u] to be the number of back-edges "passing over" the edge p→u.

The edges passing over p→ u are :

  • The edges that go up from u .
  • The edges that also pass over an edge between u and one of its children except the one where upper endpoint if u .

The first one adds the term (# of back-edges going up from u) , the second one adds (∑dp[v]), and the exception subtracts (# of back-edges going down from u).

This is how we can very easily find brides in a graph by just using the concept of DFS tree .

Articulation point :

What are articulation points ?

We say that a vertex V in a graph G with C connected components is an articulation point if its removal increases the number of connected components of G.

They are also called cut vertices . Now why should we study about them ?

Similar to bridge , Articulation point represent vulnerabilities in a graph and hence can be exploited to our benefit .

Now just how we leaned how to find bridges , in a very similar fashion we can learn how to find Articulation points .

Firstly , the root vertex of the DFS tree would be a an articulation point if and only if it has more than one child .

Now for the other vertices , if there is any subtree of a vertex u such that there is not an edge that passes over it , then the vertex would be an articulation point since on removing that vertex , that subtree and the rest of the graph becomes disconnected .

In the bridges example , we calculated dp[c]dp[c]dp[c]﻿ ( the number of back edges passing over c and it's parent ) . Now it consists of three types of edges →

  • Originating at c and ending at any of the ancestors of u.
  • Originating at any descendant of c and ending at any of the ancestors of u.
  • Originating at any descendant of c and ending at u.

Clearly we should remove the edges which are ending at u since we would loose this edge as soon as we remove vertex u . Now all we need to do is remove such edges from dp[u]dp[u]dp[u]﻿ and check if it is zero or not .

So if for any child u of c such that dp′[u]==0,dp'[u]==0,dp′[u]==0,﻿ (where dp' is new dp we defined ) , then vertex c is an articulation point .
