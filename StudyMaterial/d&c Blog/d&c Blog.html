Divide and Conquer Technique :

Your younger brother Mono-gon is trying to learn how to multiply matrices effectively but since he is very young (and naive) , the best he can do is just the school method .

That is given two matrices A and B of order N∗NN*NN∗N﻿ . The best Mono-gon is do is O(N3)O(N^3)O(N3)﻿ by simply going through all the n values of rows and columns in order to find each element of the output matrix .

The code Mono-gon wrote was :

void multiply(int A[][N], int B[][N], int C[][N])
{
    for (int i = 0; i < N; i++)
    {
        for (int j = 0; j < N; j++)
        {
            C[i][j] = 0;
            for (int k = 0; k < N; k++)
            {
                C[i][j] += A[i][k]*B[k][j];
            }
        }
    }
}

You need to help Mono-gon in order to find a faster method of multiplying two matrices . Can you do it ?

Let us first try to understand the case of 2 cross 2 matrix . In this case , we would need to do 8 separate multiplications in order to find all the elements of the output matrix .

Now we need to somehow reduce the number of multiplication calls from 8 .Like imagine if we could somehow reduce the multiplication calls from 8 to say 7 , then the complexity would reduce by a lot and as it turns out , there is indeed a way we can find the elements of the output matrix using only 7 distinct multiplication calls .

So instead of finding the values of the output matrix themselves , we found some intermediate values in terms of which we were able to represent all the terns of output matrix .

Now all we are required to use is divide and conquer which we have already gone through once in the sorting blog but let us review it once more .

A divide and conquer algorithm is a strategy of solving a large problem by :

  • Breaking the problem into smaller sub-problems.
  • Solving the sub-problems, and
  • Combining them to get the desired output.

This strategy uses recursion through which we have already gone through in the recursion and backtracking blog . Now assuming we know how to use this algorithm , let is write the final steps of our problem :

  • Divide matrices A and B in 4 sub-matrices of size N/2 x N/2 each as shown here

  • Calculate following values recursively.ae+bg,af+bh,ae+bg,af+bh,ae+bg,af+bh,﻿ ce+dg,cf+dhce+dg,cf+dhce+dg,cf+dh﻿.
  • This can be done by only using 7 recursive calls instead of 8 .
  • We can continue doing recursion until we reach the base case where the dimension of the matrix is just 1 cross 1 in which we can just return the value of that matrix .

Thus , the complexity for this algorithm would be around :

T(N) = 7T(N/2) + O(N2)

From master's theorem , time complexity would be O(NLog7) which is approximately O(N^(2.8074)).

This algorithm is also popularly known as the strassen algorithm for multiplying two matrices .

Hence we observe how a simple observation allowed us to greatly reduce the time complexity .
