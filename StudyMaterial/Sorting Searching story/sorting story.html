The task requires each of you to sort the given Algorithms in increasing order of time using different algorithms but there is one twist ! Both of these algorithms would need to be stable by nature that is you would need to preserve the original order of the input set . Formally the algorithm would be stable if two objects with equal keys appear in the same order in sorted output as they appear in the input array to be sorted.

Bubble Guppy is going to use bubble sort to sort the input (quite natural given his name ) , so your job would be to sort the input using a stable sort different than Bubble sort.

Now let's define the problem formally . Given a list of Algorithms with the corresponding time needed to learn or master these Algorithms , we need to sort them in operations fewer than what we would need if we sort them using bubble sort .

The input is as follows :

  • A_1 → 8 hrs
  • A_2 → 5 hrs
  • A_3 → 1 hrs
  • A_4 → 5 hrs
  • A_5 → 2 hrs

Let us first understand how Bubble Guppy is going to take to sort these list of numbers .

In Bubble sort , we repeatedly swap the elements if they are not in correct order . So in every pass , we try to put the largest element in the position where it should be according to the sorted array .

Let us see how the algorithm will play out →

  • 8,5,1,5,2 → swap 8 and 5
  • 5,8,1,5,2 →swap 8 and 1
  • 5,1,8,5,2 → swap 8 and 5
  • 5,1,5,8,2 → swap 8 and 2
  • 5,1,5,2,8 →swap 5 and 1
  • 1,5,5,2,8 → swap 5 and 2
  • 1,5,2,5,8 → swap 5 and 2
  • 1,2,5,5,8 → Sorted Array

This is how Bubble Guppy is going to sort the given list of input .

Now let us try to think of a way we can sort the input using a sort other than Bubble sort and which is also stable

In our previous adventure ,We learned the divide and conquer algorithm so let us try to put that into use and use Merge sort to sort the given input . In this algorithm instead of sorting the whole array at once , we divide the array into two smaller arrays of almost equal size and then after sorting these smaller arrays see if we can somehow combine the two smaller sorted arrays to form the actual array in sorted order.

Here is how merge sort uses the divide and conquer idea →

  • Divide : Divide the input array into smaller array by finding the number q of the position midway between l and r (here l and r are the index of starting and ending position ).
  • Conquer : Conquer by recursively sorting the sub-arrays in each of the two sub-problems created by the divide step.
  • Combine : Combine by merging the two sorted sub-arrays back into the single sorted sub-array A[l...r].

The base case is that for a single element we don't need to Divide since a single element is sorted by itself .

Let us see how the algorithm sorts the given input :

  • Divide into subarrays (8,5) and (1,5,2)
  • Divide (8,5) further into (8) and (5) . Now we don't need to break them further so we can combine them as (5,8) since 5 < 8 .
  • Divide (1,5,2) into (1) and (5,2) .
  • As above , (5,2) again divides into (5) and (2) and then merges to form (2,5) .
  • Now (1) and (2,5) combine as (1,2,5) since 1<2
  • Finally (5,8) and (1,2,5) merge as (1,2,5,5,8) to form the final sorted array .

The merge or combine function takes the sorted subarrays as input and then combines them in a sorted manner . It greedily tries to pick the sorted element from both the list . We are able to do this since the input sub-arrays are already in sorted order , so it does not break monotonicity .

Pseudo code would look as follows :

MergeSort(arr, left, right):
	 if left > right 
        return
    mid = (left+right)/2
    mergeSort(arr, left, mid)
    mergeSort(arr, mid+1, right)
    merge(arr, left, mid, right)
end
