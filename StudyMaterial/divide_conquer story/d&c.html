FFT :

Let as assume we have two polynomials A(x) and B(x) of say different degrees n and m respectively. Firstly we will make their degrees equal by adding terms with coefficient zero .

So now we can say both have degree zero , now let us think how to represent any general polynomial . We can just find n+1 points on the plane from where the polynomial would pass through and we would easily get the equation.

Now let's try to use odd and even degree property to break down the polynomial so that we can implement what we thought above.

The most intuitive way to break down the polynomial would be like

A(x)=A1(x2)+x.A2(x2)A(x)=A1(x^2)+x.A2(x^2)A(x)=A1(x2)+x.A2(x2)﻿

Here , A1 is the sub polynomial with even degree terms from the original polynomial and A2 is the one with odd degree terms.

This reason why we did this was because now instead of calculating d=2*n+1 points, we would only need to find d/2 points because they would occur in conjugate pairs (x,-x).

So we can easily use divide and conquer technique to solve this problem . But there is one issue , after we apply the recursion once , all points would become positive since (-x*-x=x^2) and hence our recursion would fail.

So we have to think of some way so that even after one recursion step, we would be left with +- pairs.

This can be done by introducing complex numbers!

We can choose root of unity to solve this problem .We know the roots of unity are conjugate and applying recursion would preserve the +- property.

Let us also solve one problem using the concept of divide and conquer :

Q) Find ana^nan﻿ in logarithmic time .

Solution ) We can solve this problem in O(n)O(n)O(n)﻿ by repeatedly multiplying a and so the multiplication will happen n times . But using divide and conquer approach , we can square aaa﻿ each time, rather than multiplying it with a itself. i.e If we want to compute a8a^8a8﻿ we actually compute it like

Prod = ( (a ^ 2) ^ 2) ^ 2

So we can say that if n is divisible by 2 , we can return the result of multiplying a first by itself , that is first we find a2a^2a2﻿ and then raise it to the power n/2n/2n/2﻿.

Similarly if n is odd , we can first multiply a with itself so that now n is even and repeat what we did above .

So we will be able to multiply the integers in only O(logn)O(logn)O(logn)﻿ time .

long power(int a, int n)
    {
        if(n==0)
            return 0;
        // putting a check to avoid unnecessary recursive calls
        if(a == 0)
            return 0;
        if(n%2 == 0) // b is even
            return power(a*a, b/2);
        else
            return a * power(a*a, b/2);
    }
