In this introductory blog , we will go through some of the important data structures that we would be using throughout the course of learning algorithms :

Binary trees :

A tree whose elements have at most 2 children is called binary tree .

It is a very useful data structure that can be used to do a number of things , but we don't explicitly need to implement a binary tree every time we want to use it .

It is already implemented using the set data structure in stl.

Sets are a type of associative containers in which each element has to be unique because the value of the element identifies it.

They are extremely useful and have the following properties :

  • The set stores the elements in sorted order.
  • All the elements in a set have unique values.
  • The value of the element cannot be modified once it is added to the set, though it is possible to remove and then add the modified value of that element. Thus, the values are immutable.

Red Black Trees :

A red-black tree is a kind of self-balancing binary search tree where each node has an extra bit, and that bit is often interpreted as the colour (red or black). These colours are used to ensure that the tree remains balanced during insertions and deletions.

Because of the balance of the tree , the search/insert time is reduced to O(logn)O(logn)O(logn)﻿.

Again they are already implemented as Maps in stl.

Maps are associative containers that store elements in a mapped fashion. Each element has a key value and a mapped value. No two mapped values can have same key values.

Hash Maps :

Hashing is a technique or process of mapping keys, values into the hash table by using a hash function. It is done for faster access to elements.

We have unordered_map is stl which can take use of Hash table .

The key provided to map are hashed into indices of a hash table that is why the performance of data structure depends on hash function a lot but on an average, the cost of search, insert and delete from the hash table is O(1).

Disjoint Set Union :

This is a very important data structure with two main functionalities.

We are given several elements, each of which is a separate set. A DSU will have an operation to combine any two sets, and it will be able to tell in which set a specific element is.

Sets are stored in the form of trees where each tree will correspond to one set .The root of the tree will be the representative/leader of the set.

The code for it is as follows :

void make_set(int v) {
    parent[v] = v;
}

int find_set(int v) {
    if (v == parent[v])
        return v;
    return find_set(parent[v]);
}

void union_sets(int a, int b) {
    a = find_set(a);
    b = find_set(b);
    if (a != b)
        parent[b] = a;
}

Union set is used to do union of the sets where a and b belong .

Find_set is used to return the parent of the set where v belongs .

Make_set is used to assign parent to an element .
